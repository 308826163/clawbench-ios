ClawBench iOS 3.0 原生交互架构方案
版本：v3.0
目标：在保持 ≥95% 代码复用率的前提下，实现接近原生 iOS App（Telegram、Apple Music、Safari）的交互体验，并确保后续能够低成本同步 ClawBench 上游版本更新。

================================================================================
一、设计目标
================================================================================

核心理念：

业务逻辑继续运行在 Vue 中；
交互体验逐步迁移到 SwiftUI 原生层。

实现：

高代码复用
+
原生级交互
+
低维护成本
+
可持续升级

最终体验：

✓ Telegram iOS 的连续滑动 TabBar
✓ 系统级返回手势
✓ 原生导航动画
✓ 原生键盘交互
✓ Material 毛玻璃效果
✓ Haptic 震动反馈
✓ 深色模式自动适配
✓ 120Hz ProMotion 动画
