ClawBench iOS - Claude Code 实施蓝图 v3.1
版本：v3.1
定位：架构方案 + Claude Code 执行规范 + 实施约束
目标：在保持 ≥95% 代码复用率的前提下，实现原生 iOS 交互体验，并严格约束 Claude Code 按方案执行，避免幻觉式重构和偏离设计目标。

================================================================================
一、项目核心目标
================================================================================

第一目标：
最大限度复用现有代码资产。

第二目标：
获得接近原生 App 的交互体验。

第三目标：
确保能够长期同步 ClawBench 上游更新。

第四目标：
禁止 Claude Code 自行理解并重构架构。

核心理念：

业务属于 Vue；
交互属于 SwiftUI；
Claude 属于执行层；
架构决策权永远属于人。

================================================================================
二、总体架构
================================================================================

Go Backend
        │
Vue3 + TypeScript
        │
Capacitor Bridge
        │
SwiftUI Native Shell
        │
iOS Framework

职责划分：

Go：
负责全部业务能力。

Vue：
负责全部业务页面和状态管理。

SwiftUI：
负责全部交互体验和系统能力。

================================================================================
三、代码复用目标
================================================================================

Go 后端：
100%

Vue 页面：
95%

业务逻辑：
98%

资源文件：
100%

新增原生层：
约 5%

禁止为了获得原生体验而大规模重写业务代码。

================================================================================
四、Claude Code 第一原则（最高优先级）
================================================================================

Claude Code 是执行者，不是架构设计者。

禁止：

❌ 自行修改架构
❌ 自行新增依赖
❌ 自行迁移目录
❌ 自行重命名模块
❌ 自行替换技术方案
❌ 自行推断需求
❌ 自行重写页面
❌ 自行优化未授权模块

允许：

✅ 严格按照方案执行
✅ 输出风险提醒
✅ 输出备选方案
✅ 请求人工确认

================================================================================
五、执行优先级
================================================================================

Priority 1
遵守实施蓝图。

Priority 2
遵守现有代码结构。

Priority 3
最小修改原则。

Priority 4
保证可回滚。

任何情况下：
前一优先级高于后一优先级。

================================================================================
六、架构冻结原则
================================================================================

以下目录属于冻结区域：

internal/
cmd/
config/
web/src/
web/css/
assets/

禁止：

重命名
迁移
拆分
重构
替换实现

允许：

新增文件
新增桥接接口
新增适配代码

原则：

原页面能运行，就不要重写。

================================================================================
七、Vue 层约束
================================================================================

Vue：

负责：

页面渲染
状态管理
组件逻辑
接口调用

禁止：

❌ 重写页面
❌ 重构 Store
❌ 重写 Router
❌ 修改 Agent 实现
❌ 修改业务流程

原则：

业务逻辑全部保留。

================================================================================
八、SwiftUI 层约束
================================================================================

SwiftUI：

只允许接管：

Navigation
TabBar
Sheet
ContextMenu
Keyboard
Share
File Picker
Haptic
Notification

禁止：

❌ 保存业务状态
❌ 实现业务逻辑
❌ 调用业务 API
❌ 修改 Vue 状态树

原则：

SwiftUI 是交互容器，
不是业务容器。

================================================================================

禁止：

多个功能混合提交。

十五、回滚规范
================================================================================

所有任务：

必须提供：

回滚命令：

或者：

git revert <commit-id>

任何修改：
必须可回滚。
